## 1. 产品概述

### 1.1 产品定位

MCP 服务中台是一个基于 Model Context Protocol (MCP) 协议的服务集成平台，旨在将传统系统的 OpenAPI 接口快速转换为 MCP 服务，使 AI 应用能够无缝调用现有企业服务。

### 1.2 目标用户

- **AI 应用开发者**：需要快速集成后端服务的 AI 能力
- **企业 IT 部门**：已有大量 OpenAPI 服务，需要安全暴露给 AI 应用
- **SaaS 服务提供商**：希望将服务接入 MCP 生态的厂商

### 1.3 核心价值

| 价值维度 | 说明 |
|---------|------|
| **协议桥接** | 将 RESTful/OpenAPI 等传统 API 协议透明转换为 MCP 协议 |
| **统一治理** | 提供集中的服务注册、发现、认证、限流等治理能力 |
| **快速接入** | 通过配置化方式快速将现有 OpenAPI 暴露为 MCP 服务 |
| **安全隔离** | 在 AI 模型与后端服务之间建立安全边界 |

```
          ▼
┌───────────────────┐

│   - 转换为 Tool    │
│   - 构建 Schema    │
│   - 注册 Handler   │
└─────────┬─────────┘
          │
          ▼
┌───────────────────┐
│   ToolRegistry    │
│   - 工具注册表    │
│   - 调用路由      │
└───────────────────┘
```

#### 3.3.2 限流实现

基于 Redis 的滑动窗口限流算法：

```java
public class RateLimiter {
    public boolean tryAcquire(String key, int limit, int windowSeconds) {
        Long count = redis.opsForValue().increment(key);
        if (count == 1) {
            redis.expire(key, windowSeconds, TimeUnit.SECONDS);
        }
        return count <= limit;
    }
}
```

## 5. 安全设计

### 5.1 认证机制

```
┌─────────────────┐
│  MCP Client     │
│  (AI Agent)     │
└────────┬────────┘
         │ 1. 携带 API Key
         ▼
┌─────────────────┐
│  API Gateway    │
│  - 验证 API Key  │
│  - 检查权限     │
│  - 限流检查     │
└────────┬────────┘
         │ 2. 转发请求
         ▼
┌─────────────────┐
│  Backend Service│
│  (OpenAPI)      │
└─────────────────┘
```

### 5.2 限流策略

| 策略类型 | 说明 | 默认值 |
|---------|------|--------|
| 全局限流 | 限制整个系统的请求量 | 100 次/分钟 |
| 服务限流 | 限制单个服务的请求量 | 可配置 |
| 客户端限流 | 限制单个 API Key 的请求量 | 可配置 |

## 8. 风险与挑战

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| MCP 协议变更 | 高 | 关注官方动态，设计可扩展的协议层 |
