## 1. 快速开始

### 1.1 一分钟启动

```bash
# 1. 克隆项目
git clone https://github.com/HammCn/mcp-platform.git
cd mcp-platform

# 2. 初始化数据库
mysql -u root -p < database/schema.sql

# 3. 启动 Redis
redis-server

# 4. 编译并启动
mvn clean install
mvn spring-boot:run -pl mcp-admin
```

### 1.2 验证启动

访问以下地址：
- **Swagger UI**: http://localhost:8080/swagger-ui.html
- **健康检查**: http://localhost:8080/actuator/health

## 3. 安装部署

### 3.1 数据库初始化

```bash
# 方式 1: 命令行执行
mysql -u root -pasdfew12345 < database/schema.sql

# 方式 2: MySQL Workbench
# 打开 database/schema.sql 文件并执行
```

### 3.2 编译打包

```bash
# 编译所有模块
mvn clean install -DskipTests

# 打包为 JAR
cd mcp-admin
mvn package -DskipTests

# JAR 文件位置
# mcp-admin/target/mcp-admin-1.0.0-SNAPSHOT.jar
```

### 3.3 启动方式

#### 方式 1: Maven 启动（开发环境）

```bash
mvn spring-boot:run -pl mcp-admin
```

#### 方式 2: JAR 包启动（生产环境）

```bash
java -jar mcp-admin/target/mcp-admin-1.0.0-SNAPSHOT.jar
```

#### 方式 3: Docker 启动（需自行构建镜像）

```bash
docker build -t mcp-platform .
docker run -p 8080:8080 mcp-platform
```

### 3.4 后台运行

```bash
# 使用 nohup
nohup java -jar mcp-admin.jar > app.log 2>&1 &

# 使用 systemd（Linux）
sudo systemctl start mcp-platform
```

## 4. 配置说明

### 4.1 核心配置

配置文件位置：`mcp-admin/src/main/resources/application.yml`

```yaml
server:
  port: 8080  # 服务端口

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mcp_platform
    username: root
    password: your_password  # 修改为实际密码
  
  data:
    redis:
      host: localhost
      port: 6379
      password:  # Redis 密码（无密码留空）

mcp:
  server:
    name: MCP Platform  # MCP 服务器名称
    version: 1.0.0

rate-limit:
  enabled: true
  default-limit: 100  # 默认限流次数
  default-window: 60  # 默认时间窗口（秒）

auth:
  api-key:
    enabled: true
    header-name: X-API-Key
  jwt:
    secret: your-secret-key  # 生产环境请修改
```

### 4.2 环境变量配置

生产环境建议使用环境变量：

```bash
export DB_HOST=localhost
export DB_PORT=3306
export DB_USERNAME=root
export DB_PASSWORD=your_password
export REDIS_HOST=localhost
export REDIS_PORT=6379
export JWT_SECRET=your-secret-key

# 启动应用
java -jar mcp-admin.jar
```

### 4.3 生产环境配置

使用 `application-prod.yml`：

```bash
java -jar mcp-admin.jar --spring.profiles.active=prod
```

## 5. 服务管理

### 5.1 创建服务

```bash
curl -X POST "http://localhost:8080/api/admin/services" \
  -H "Content-Type: application/json" \
  -d '{
    "serviceCode": "user-service",
    "serviceName": "用户服务",
    "description": "用户管理相关 API",
    "version": "1.0.0",
    "serviceType": "HTTP",
    "baseUrl": "http://localhost:8081",
    "status": "REGISTERED",
    "mcpEnabled": true,
    "mcpToolPrefix": "user"
  }'
```

### 5.2 注册服务实例

```bash
curl -X POST "http://localhost:8080/api/admin/instances" \
  -H "Content-Type: application/json" \
  -d '{
    "serviceId": 1,
    "instanceAddress": "192.168.1.100:8081",
    "weight": 100,
    "metadata": {
      "zone": "beijing",
      "version": "1.0.0"
    }
  }'
```

### 5.3 启用/禁用服务

```bash
# 启用服务
curl -X POST "http://localhost:8080/api/admin/services/1/enable"

# 禁用服务
curl -X POST "http://localhost:8080/api/admin/services/1/disable"
```

### 5.4 刷新服务工具

当 OpenAPI 规范更新后：

```bash
curl -X POST "http://localhost:8080/api/admin/services/1/refresh"
```

## 6. API Key 管理

### 6.1 创建 API Key

```bash
curl -X POST "http://localhost:8080/api/admin/api-keys" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "测试 API Key",
    "description": "用于测试的 API Key",
    "owner": "admin",
    "allowedServices": ["user-service"]
  }'
```

响应示例：
```json
{
  "code": "SUCCESS",
  "data": {
    "id": 1,
    "apiKey": "mk_abc123def456...",
    "name": "测试 API Key",
    "status": "ACTIVE"
  }
}
```

### 6.2 使用 API Key

在请求头中添加 API Key：

```bash
curl -X GET "http://localhost:8080/api/admin/services" \
  -H "X-API-Key: mk_abc123def456..."
```

### 6.3 禁用/启用 API Key

```bash
# 禁用
curl -X POST "http://localhost:8080/api/admin/api-keys/1/disable"

# 启用
curl -X POST "http://localhost:8080/api/admin/api-keys/1/enable"
```

## 7. MCP 协议使用

### 7.1 建立 SSE 连接

```bash
# 使用 curl 连接
curl -N http://localhost:8080/mcp/sse
```

响应示例：
```
event: endpoint
data: /mcp/message?sessionId=abc123
```

### 7.2 发送 Initialize 请求

```bash
curl -X POST "http://localhost:8080/mcp/message?sessionId=abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {
      "protocolVersion": "2024-11-05",
      "capabilities": {},
      "clientInfo": {
        "name": "test-client",
        "version": "1.0.0"
      }
    }
  }'
```

### 7.3 获取 Tools 列表

```bash
curl -X POST "http://localhost:8080/mcp/message?sessionId=abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/list"
  }'
```

### 7.4 调用 Tool

```bash
curl -X POST "http://localhost:8080/mcp/message?sessionId=abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 3,
    "method": "tools/call",
    "params": {
      "name": "user_get_user",
      "arguments": {
        "userId": 123
      }
    }
  }'
```

## 8. OpenAPI 转换

### 8.1 准备 OpenAPI 规范

创建 `openapi.yaml`：

```yaml
openapi: 3.0.0
info:
  title: 用户服务 API
  version: 1.0.0
servers:
  - url: http://localhost:8081
paths:
  /users/{userId}:
    get:
      operationId: getUser
      summary: 获取用户信息
      parameters:
        - name: userId
          in: path
          required: true
          schema:
            type: integer
      responses:
        '200':
          description: 成功
```

### 8.2 创建服务时指定 OpenAPI

```bash
curl -X POST "http://localhost:8080/api/admin/services" \
  -H "Content-Type: application/json" \
  -d '{
    "serviceCode": "user-service",
    "serviceName": "用户服务",
    "serviceType": "HTTP",
    "baseUrl": "http://localhost:8081",
    "specType": "CONTENT",
    "openapiSpec": "openapi: 3.0.0\n...",
    "mcpEnabled": true,
    "mcpToolPrefix": "user"
  }'
```

### 8.3 自动生成的 MCP Tools

系统会自动将 OpenAPI 路径转换为 MCP Tools：

| OpenAPI 路径 | MCP Tool 名称 |
|-------------|--------------|
| GET /users/{userId} | user_getUser |
| POST /users | user_createUser |
| PUT /users/{userId} | user_updateUser |
| DELETE /users/{userId} | user_deleteUser |

## 10. 常见问题

### 10.1 启动问题

#### Q: 数据库连接失败

**A:** 检查以下几点：
1. MySQL 是否启动：`mysql -u root -p`
2. 数据库是否创建：`SHOW DATABASES;`
3. 用户名密码是否正确

#### Q: Redis 连接失败

**A:** 
```bash
# 检查 Redis 状态
redis-cli ping

# 启动 Redis
redis-server

# macOS 使用 brew
brew services start redis
```

#### Q: 端口被占用

**A:** 修改 `application.yml`：
```yaml
server:
  port: 8081  # 改为其他端口
```

### 10.2 编译问题

#### Q: Maven 依赖下载失败

**A:** 使用阿里云镜像，已在 `pom.xml` 中配置

#### Q: Java 版本不兼容

**A:** 确保使用 JDK 17+：
```bash
java -version
```

### 10.3 MCP 协议问题

#### Q: SSE 连接断开

**A:** 检查网络和防火墙设置

#### Q: Tool 调用失败

**A:** 
1. 检查服务实例是否已注册
2. 检查 OpenAPI 规范是否正确
3. 查看请求日志排查问题

### 10.4 获取帮助

- **GitHub Issues**: https://github.com/HammCn/mcp-platform/issues
- **技术文档**: docs/ 目录
- **API 文档**: http://localhost:8080/swagger-ui.html
